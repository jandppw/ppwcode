Invariants

Type invariants express conditions that have to hold for all objects of the type (and subtypes) at stable times (i.e., inbetween calls of public methods). Behavioral subtyping says that in subtypes invariants can be added or strengthened, but not weakened or removed.

Public type invariants are mentioned as {@link Expression expressions} inside a {@link TypeContract} annotation on the type level, or, since some invariants are particulary focussed on one property, inside the {@link Basic} annotation ({@link Basic#invars()}) on the basic inspector of the property.
